ChunkIt

Crop, Extract, De-sprite and Play

View on GitHub Download .zip Download .tar.gz

Welcome words!

A jQuery plugin and image processing utility. This is useful when you are playing with images. This enables you to extract portion of image, crop, de-sprite and many more.

Browser Requirements

This plugin is using HTML5 canvas internally. Hence, It require IE9 or higher version and latest Mozilla, Chrome. Please read following url for browser support

Options

imgURL

Type: String | Default: ''

This is act as a source for ChunkIt. It can be either remote url or base64 encoded data url. This is used to feed image data. Alternative, User can feed image url from DOM elements. It reads data-chunkit-imgurl="images/<img-name>".

Code Example:

$('.selector').chunkIt({ imgURL : 'images/someImage.jpg' }); // OR // ChunkIt url in inline <div class="selector" data-chunkit-imgurl="images/shuffle.jpg" ></div> $('.selector').chunkIt();

maxContainerWidth

Type: Number | Default: 0

This option specifies grid or table maximum possible width. This is used to adjust the cropped image cells width to fit container. It is doing intelligent calculation based on number of cells in a row and width of each cells to make perfect grid option.

Code Example:

$('.selector').chunkIt({ maxContainerWidth: 300 /* equivalent to 300px */ });

scaleImage

Type: Boolean | Default: false

This is set to automatically adjust maximum width of image container(maxContainerWidth) based on image width. If this options is true, then maxContainerWidth become disabled internally.

Code Example:

$('.selector').chunkIt({ scaleImage: true });

startXY

Type: Array | Default: [0, 0] -> [top, left]

Sets starting point to crop image. The default indicates the left-top corner of the image. It can be vary from [0, 0] to [image-width, image-height]. Array should contain any positive number. Any invalid or negative automatically turns into [0, 0].

Code Example:

$('.selector').chunkIt({ startXY: [50,50] /* equivalent to [50px from top, 50px from left] */ });

endXY

Type: Array | Default: [0, 0] is equivalent to [<image width>, <image height>]

This option sets end point to crop image. The default indicates the right-bottom corner of the image. It can be vary from [, ] to [, ]. Array should contain any positive number. Any invalid or negative automatically turns into [<image width>, <image height>].

Code Example:

$('.selector').chunkIt({ endXY: [50,50] /* equivalent to 50px from top and 50px from left */ });

cellsInRow

Type: Number | Default: 5

This option specifies number of cell in a row. This is directly affect cells width. Default option forces the cell width to take image’s width.

Code Example:

$('.selector').chunkIt({ cellsInRow: 5});

cellsInColumn

Type: Number | Default: 5

This option specifies number of cell in a row. This is directly affect cells width. Default option forces the cell width to take image’s width.

Code Example:

$('.selector').chunkIt({ cellsInColumn: 5});

shrinkCellDimension

Type: Boolean | Default: true

This option forces endXY to image’s width and height when endXY set greater than image dimension. This option avoids excess white space in right and bottom side of cropped region. This is internally override the option cropExtraSpaces:true. This option ensures all the cropped cells dimensions are equal.

Code Example:

$('.selector').chunkIt({ shrinkCellDimension: true });

cropExtraSpaces

Type: Boolean | Default: false

This option forces endXY to image’s width and height when endXY set greater than image dimension. Unlike shrinkCellDimension, it is adjusting cells dimension which are in image edge. That is, cells are cropped from right and bottom portion of image.

Code Example:

$('.selector').chunkIt({￼￼￼ cropExtraSpaces: true });

shuffle

Type: Boolean | Default: false

This option plots cropped cell in random order. This is optional and creative option. And it is recommended to use when shrinkCellDimension is set with true.

Code Example: $('.selector').chunkIt({ shrinkCellDimension: true /* This option should be true shuffle: true */ });

gridCellWrapperTpl

Type: String | Default: '<li class="cell"></li>'

Provide helper functionality to customized HTML element wrapper template. Default option creates list tag wrapped around cropped image portion.

Code Example:

$('.selector').chunkIt({ gridCellWrapperTpl: '<div class=”customClass” customAttribute=”attrVal” ></div>' });

gridRowWrapperTpl

Type: String | Default: ''

Provide helper functionality to customized HTML element wrapper template. It creates the parent HTML element for gridCellWrapperTpl templates along with cropped image portions. ￼￼ Code Example:

$('.selector').chunkIt({ gridRowWrapperTpl: '<p class=”customClass” customAttribute=”attrVal” ></p>' });

gridBodyWrapperTpl

Type: String | Default: '<ul class="grid"></ul>'

Provide helper functionality to customized HTML element wrapper template. It creates the parent HTML element for gridRowWrapperTpl templates along with cropped image portions.

Code Example:

$('.selector').chunkIt({ gridBodyWrapperTpl: '

' });

Callbacks

onImageLoad : function($event, currentObj, $imageElem)

This callback function will be executed after image load.

Code Example

$('.selector').chunkIt({ onImageLoad: function($event, currentObj, $imageElem){ console.log($imageElem.attr('src')); } });

onGetGridMetaData : function($event, currentObj, gridData)

This callback function will be executed after generating cropping co-ordinates.

onShuffleCoordinates : function($event, currentObj, gridData)

This callback function will be executed on after shuffle done.

onCellWrapping : function($event, currentObj, $wrappedCellElem, $wrappedParentElem)

This callback function will be executed on wrapping cell DOM element by using wrapper template.

onRowWrapping : function($event, currentObj, $wrappedRowElem, $wrappedParentElem)

This callback function will be executed on wrapping row DOM elements by using wrapper template.

onGetGridElem : function($event, currentObj, $gridElem)

This callback function will be executed on generating grids DOM element.

onAfterFinish : function($event, currenObj)

This callback function will be executed on after inserting grid in document object.

Events

Events are respective to callback. In real, the callbacks in option are interlly attached events. The only difference is, option define callbacks takes precedence than externall attached events.

imageLoad

This imageLoad event will be triggered after image load.

Code Example

$('.selector').on('imageLoad', function($event, currentObj, $imageElem){ console.log($imageElem.attr('src')); });

getGridMetaData

This getGridMetaData event will be triggered after generating cropping co-ordinates.

shuffleCoordinates

This shuffleCoordinates event will be triggered on after shuffle done.

cellWrapping

This cellWrapping event will be triggered on wrapping cell DOM element by using wrapper template.

rowWrapping

This rowWrapping event will be triggered on wrapping row DOM elements by using wrapper template.

getGridElem

This getGridElem event will be triggered on generating grids DOM element.

afterFinish

This afterFinish event will be triggered on after inserting grid in document object.
